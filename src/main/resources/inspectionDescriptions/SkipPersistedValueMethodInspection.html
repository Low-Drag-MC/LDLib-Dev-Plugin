Validates that methods annotated with @SkipPersistedValue correctly reference fields and have proper signatures.

The inspection verifies that the method returns boolean, has exactly one parameter matching the field type, and references a field annotated with @Configurable.

The @SkipPersistedValue annotation is used to conditionally skip persistence of field values. The annotated method should return true if the value should not be persisted.

Example (correct usage):


@Configurable
int intField = 10;

@SkipPersistedValue(field = "intField")
public boolean skipIntFieldPersisted(int value) {
    return value == 10;
}

Problems detected:

  • Field specified in field attribute doesn't exist
  • Referenced field is not annotated with @Configurable
  • Method doesn't return boolean
  • Method has wrong number of parameters (must have exactly one)
  • Parameter type doesn't match the field type
  • Method is declared as static

This inspection helps ensure that skip persistence conditions are properly configured for synchronized data fields.
