Detects missing or incorrectly defined UpdateListener methods for fields annotated with @UpdateListener.

This inspection validates that for each field annotated with @UpdateListener, there exists a corresponding method with the specified name and correct signature. The method must have exactly two parameters of the same type as the annotated field (or its supertype).

Example of correct usage: 


@UpdateListener(methodName = "onValueChanged")
private int value;

private void onValueChanged(int oldValue, int newValue) {
    // Handle value change
}

This inspection will report an error if:

  • The specified method does not exist
  • The method has incorrect number of parameters (not exactly 2)
  • The parameter types don't match the field type

This inspection helps ensure that UpdateListener annotations are properly configured and their corresponding listener methods are correctly implemented.
