Detects missing or incorrectly defined ConfigSearch search configurator methods for fields annotated with @ConfigSearch.

This inspection validates that for each field annotated with @ConfigSearch that specifies a searchConfiguratorMethod, there exists a corresponding method with the specified name and correct signature. The search configurator method must have no parameters and return ISearchConfigurator.

Example of correct usage: 


@ConfigSearch(searchConfiguratorMethod = "createSearchConfigurator")
private Block block = Blocks.STONE;

private ISearchConfigurator createSearchConfigurator() {
    return new ISearchConfigurator() {
        // Implement search configurator logic
    };
}

This inspection will report an error if:

  • The specified search configurator method does not exist
  • The method has parameters (should be parameterless)
  • The method doesn't return ISearchConfigurator
  • The method is static (should be non-static)

This inspection helps ensure that ConfigSearch annotations are properly configured and their corresponding methods are correctly implemented for searching or configuring.
