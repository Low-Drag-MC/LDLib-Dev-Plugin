Detects incompatible field types for configuration annotations.

This inspection validates that fields annotated with specific configuration annotations have compatible types as required by the annotation contract.

Type requirements:

  • @ConfigColor: Field must be int or Integer
  • @ConfigHDR: Field must be Vector4f
  • @ConfigList: Field must be an array or Collection type
  • @ConfigSelector: Field must be an enum or String

Example of correct usage: 


@ConfigColor
private int backgroundColor = 0xFFFFFF;

@ConfigHDR
private Vector4f lightDirection = new Vector4f();

@ConfigList
private List<String> items = new ArrayList<>();

@ConfigSelector
private String mode = "default";

@ConfigSelector
private MyEnum option = MyEnum.OPTION1;

This inspection will report a warning when:

  • A field type doesn't match the expected type for the annotation
  • The field type is incompatible with the annotation's functionality

This inspection helps ensure that configuration annotations are applied to fields with appropriate types, providing clear warnings about type mismatches.
