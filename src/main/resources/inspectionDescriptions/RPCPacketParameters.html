Reports calls to RPCPacketDistributor where the provided arguments do not match the parameters of the target @RPCPacket method.

The inspection checks for:

  • Argument Count: The number of arguments passed to the distributor must match the number of parameters in the target method (excluding the optional RPCSender).
  • Type Compatibility: Each argument type must be compatible with the corresponding parameter type in the target method.

Note: If the target @RPCPacket method declares RPCSender as its first parameter, it is automatically injected by the system and should not be included in the distributor call arguments.

Example:


// Target Method
@RPCPacket("test")
public static void handler(RPCSender sender, String msg, int count) { ... }

// Correct Call
RPCPacketDistributor.rpcToServer("test", "Hello", 1);

// Incorrect Call (Count Mismatch)
RPCPacketDistributor.rpcToServer("test", "Hello");
